Query options

The most relevant query settings.

You can navigate to the query options in two ways.
  • From Excel: tab Data > Get Data (group Get & Transform Data) > Query Options
  • From Query Editor: File > options and settingsQuery Options

Only the most relevant settings will be covered. And several settings are self-explanatory.

GLOBAL Data Load

Using one of the aforementioned methods you get the following window.

It is important to distinguish between Global and Current Workbook settings:
  • GLOBAL: settings will affect all the projects you create.
  • CURRENT WORKBOOK: settings will affect only the current Excel file.

Default Query Load Settings

The data load settings affect how the queries are loaded. The standard load settings are
  • When loading a single query the query is loaded to a new worksheet.
  • When loading multiple queries the queries are loaded to the Data Model.

A preferred method is to load a query manually. In that case, you select Specify custom default load settings and unticking both choices. Then only a connection is built and you need to load the query manually:

  • In Excel: tab Data > Queries & Connection > Right click on query > Load to
  • In Query Editor: tab Home > Close & Load > Load to

Fast Data Load

By enabling the Fast Data Load option, your queries will take less time to load, but Excel may be unresponsive for long periods.

Data Cache Management Options

Copies of queries preview results are stored on your local disk for faster viewing later. You can clear this cache or configure how much disk space is allowed to be used.

Tip: Cleaning the cache can be useful when something is not working as expected.

Global Privacy

When you are combining data from various sources then you may get lots of error messages because these actions can conflict with your privacy level settings. These messages mostly make no sense to the end-user. To avoid these messages you can safely select Always ignore Privacy Level settings.

Current Regional Settings

The regional settings affect how Power Query manipulates dates, currencies, and other features specific to regions & countries.
